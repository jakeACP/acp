Build an Admin Panel feature called “Districts.”

Purpose:
Create, manage, visualize, and assign political/geographic districts at every level so ACP can accurately connect users, candidates, offices, elections, and representatives by location.

Routes:
- /admin/districts
- /admin/districts/new
- /admin/districts/:districtId
- /admin/districts/:districtId/edit

Core Goal:
Admins need a map-based system to define district borders, confirm accuracy, and use those districts throughout the ACP platform.

District Types:
Support districts at multiple levels:
- Federal Congressional District
- State Senate District
- State House District
- County
- City
- Township
- School Board District
- Judicial District
- Ward
- Precinct
- Custom District

AI Draft District Tool:
Add button:
“Draft Boundary with AI”

Admin provides:
- District name
- District type
- State
- County/city if applicable
- Optional source URL or pasted description

The AI should:
- Search or interpret official district boundary information if available
- Draft a GeoJSON polygon
- Set confidence score
- Mark status as “needs_review”
- Add source citation if available

Important:
AI-drafted boundaries are NEVER automatically confirmed.
Admins must review and click “Confirm District.”

Manual Review Flow:
1. AI drafts or admin imports boundary
2. District status = needs_review
3. Admin reviews map
4. Admin edits boundary if needed
5. Admin clicks “Confirm District”
6. Confirmed district becomes available across ACP

Accuracy Requirements:
- Prefer official government boundary data when available
- Store source URL and source name
- Show confidence score
- Show warning if no official source is attached
- Never use unverified AI boundaries as confirmed data

Address/User Matching:
When a user adds or updates their address:
- Geocode the address to latitude/longitude
- Check which confirmed district polygons contain that point
- Link matching districts to user profile
- Store match method and confidence score

User Profile Integration:
Show user’s matched districts:
- Congressional district
- State senate district
- State house district
- County
- City

- Ward/precinct when available

Candidate Integration:
When a candidate runs for office:
- Candidate must select an office
- Candidate must select a confirmed district
- Candidate becomes visible to users inside that district
- Candidate profile displays district map

Search People Within Borders:
Add admin tool:
“Search Users in District”

Function:
- Select confirmed district
- Query all users whose saved geocoded location falls inside boundary
- Display matched users
- Export CSV option for admins
- Respect user privacy settings

Privacy:
- Do not expose exact user addresses publicly
- Public views should only show district-level matching
- Admin-only access required for address-level tools

Show:

- Current representatives
- Related elections

Do NOT show:

Recommended Spatial Logic:
Use PostGIS if available.
If not available, store GeoJSON and use Turf.js for point-in-polygon matching.

Preferred:
- PostgreSQL + PostGIS
- geojson_boundary stored as geometry or JSONB
- spatial index for fast matching

Fallback:
- GeoJSON stored as JSON
- Turf.js booleanPointInPolygon

Admin Permissions:
Only admins can:
- Create districts
- Edit boundaries
- Confirm districts
- Archive districts
- Search users inside districts

Regular users can:
- View public district pages
- See their own districts
- See candidates in their district

- Admin can draw/edit a district boundary
- Admin can import/export GeoJSON

- Admin can search users within a district
