Replit Agent Prompt — ACP Roles, Elections, and Privileges

Goal: Implement a production-ready backend + minimal admin UI for the ACP app’s role hierarchy, elections, trust/merit scoring, transparent moderation, and privilege gating. The platform must be volunteer-run: higher roles are earned via behavior, verified identity (when required), and democratic elections. No paid roles.

Tech Stack (use defaults if not specified):
	•	Backend: Node.js (TypeScript) + Express
	•	DB: PostgreSQL + Prisma
	•	Auth: JWT + email/OTP; optional ID verification flag
	•	Queue: BullMQ + Redis
	•	Tests: Vitest
	•	Infra: Docker compose (api, db, redis)
	•	UI (minimal ops console): Next.js (TypeScript) + Tailwind (admin-only)

⸻

1) Data Model (Prisma)

Implement these tables with indexes and foreign keys:
	•	User(id, handle, email, isVerified:boolean, kycLevel:enum(None|Basic|GovID), createdAt)
	•	Role(id, key, name, tier:int, track:enum(Media|Analysis|Political|Governance|Technical|Watchdog|General), isElective:boolean, requiresKYCLevel, minScore:int, description)
	•	UserRole(userId, roleId, status:enum(Active|Revoked|Expired|Pending), grantedBy, grantedAt, expiresAt, reason)
	•	Permission(id, key, description)
	•	RolePermission(roleId, permissionId) // many-to-many
	•	Reputation(userId, truthScore:float, civilityScore:float, contributionScore:float, corruptionFighterScore:float, lastUpdatedAt)
	•	EventLog(id, actorUserId, action, targetType, targetId, metadata JSONB, createdAt) // append-only transparency log
	•	Election(id, roleId, scope:enum(Global|Region|IssueArea), nominationsOpenAt, votingOpenAt, votingCloseAt, status, method:enum(Approval|RankedChoice|Plurality))
	•	Candidate(id, electionId, userId, statement, approved:boolean)
	•	Ballot(id, electionId, voterId, data JSONB, createdAt) // store encrypted/hashed ballot data
	•	ModerationAction(id, moderatorId, targetType, targetId, actionType, reason, createdAt) // also mirrored into EventLog
	•	Content(id, authorId, type, issueTags string[], text, mediaUrl, createdAt) // for basic posts, debates, reports
	•	FactCheck(id, contentId, checkerId, verdict:enum(True|False|Mixed|Unsubstantiated), sources JSONB, confidence:0..1, createdAt)

Add necessary composite indexes for performance (e.g., UserRole(userId, roleId), Reputation(userId), Election(roleId, status)).

4) Reputation & Advancement

Implement a periodic job to compute Reputation:
	•	truthScore: based on fact-check outcomes on authored content; up for corroborated posts, down for flagged misinformation.
	•	civilityScore: community reports (weighted), moderator decisions, tone heuristics (lightweight).
	•	contributionScore: accepted proposals, hosted debates, volunteer hours, curated content featured.
	•	corruptionFighterScore: verified watchdog actions (exposed donations, FOIA docs, whistleblower validations).

Advancement Rules:
	•	Auto-eligibility when isVerified meets role’s requiresKYCLevel and sumScores ≥ minScore.
	•	If isElective = true, user becomes eligible candidate for the next election cycle for that role.
	•	Revocation: Dropping below thresholds, severe moderation hits, or auditor findings → UserRole.status = Revoked (and auto-log to EventLog).

⸻

5) Elections

Implement generic elections for any Role.isElective = true.
	•	Methods supported: Approval Voting (default), Ranked Choice (IRV), Plurality.
	•	Phases: Nominations → Vetting (KYC/eligibility) → Voting → Results → Automatic role grant.
	•	Prevent conflicts: party_admin, algorithm_council, party_auditor cannot be concurrent with candidate or elected_representative.
	•	Add regional/issue scopes using Election.scope.

APIs:
	•	POST /elections (admin/council)
	•	POST /elections/:id/nominations
	•	POST /elections/:id/ballots (enforce one-person-one-vote via auth + anti-sybil checks)
	•	GET /elections/:id/results (publish anonymized tallies + cryptographic proofs where possible)

⸻

6) Transparency & Moderation

	•	ModerationAction entries mirrored to EventLog; public ledger API:
	•	GET /ledger?type=moderation (redact victim PII where required)
	•	Changelog endpoint for code/config updates by developer and algorithm_council.
