political_lean FLOAT DEFAULT 0.0, -- trailing average in [-1,1], optional

## Feed Logic

### 1) Following (rename “Latest” → “Following”)

* **Definition:** All posts from accounts the viewer follows, ordered `created_at DESC`, cursor by `(created_at, id)`.
* **No algorithmic re-ordering.** Only filter out `is_deleted = true` and hard-blocked/hidden accounts if that exists.
* **Acceptance tests:**

  * When A follows B and C, A sees B/C posts newest first.
  * Unfollow removes future posts from that author.
  * No personalization, no post reshuffle between requests for same cursor window.

### 2) All

* **Definition:** Mixed content (users, orgs, events, news) ranked by engagement + recency with light anti-spam.
* **Engagement score (per post p):**

  ```
  base = 1*likes + 3*comments + 4*shares + 0.5*emojis + 1.5*gifs + 4*bookmarks
  penalty = 6*confirmed_flags + 1*unreviewed_flags
  score_engagement = max(base - penalty, 0)
  ```
* **Time decay:** `decay = exp(- age_hours / 18)` (tuneable)
* **Author spam cap:** at most 1 post per author per 8 items; apply small penalty if an author is overrepresented in the current window.
* **Final score:** `rank_all = score_engagement * decay * quality_boost`

  * `quality_boost` default 1.0; reserve for future signals (e.g., completion rate on long reads).
* **Acceptance tests:**

  * Higher-engagement but much older content shouldn’t dominate fresh high-quality content.
  * Posts with many flags sink quickly or get removed if threshold exceeded.

### 3) News (with Bias Meter + Monetization)

* **Per-post votes:** count `N = neutral`, `L = left`, `R = right`, `T = N+L+R`.
* **Bias index:** `bias_index = (R - L) / max(T,1)` in \[-1,1].
* **Neutrality ratio:** `neutral_ratio = N / max(T,1)`.
* **Confidence (Wilson score on neutral ratio)** to avoid tiny-sample noise. Require `T >= 20` for confident labeling; else show “Unrated”.
* **Neutrality index (ranking multiplier):**

  ```
  neutrality_index = neutral_ratio * (1 - abs(bias_index))
  ```

  This rewards many Neutral votes and penalizes lopsided Left/Right votes.
* **News ranking:**

  ```
  base_news = log1p(score_engagement)        -- same engagement as All, but log dampened
  decay = exp(- age_hours / 24)
  rank_news = base_news * decay * neutrality_index
  ```
* **Monetization policy:**

  * Monetize when `T >= 20` AND `neutrality_index >= 0.50` → `monetized = true`
  * De-monetize when `T >= 20` AND `abs(bias_index) >= 0.60` OR `neutrality_index < 0.35`
  * Unrated (`T < 20`) → show, but neither boost nor monetize.
* **Anti-brigading safeguards:**

  * One vote per user per post (enforced by unique index).
  * Require account age ≥ 7 days or verified email/phone to cast a bias vote.
  * Optional soft trust weights: weight votes by `log1p(account_age_days)` (keep simple at first; implement later).
* **Acceptance tests:**

  * Highly Neutral posts surface above biased posts of similar engagement.
  * A post can move from Unrated → Monetized as it accrues votes.
  * Biased posts remain visible but rank lower and are de-monetized.

## Endpoints (example contracts)

```http
GET /api/feed/all?cursor=<opaque>&limit=25&viewerId=abc
→ { items: PostCardDTO[], nextCursor }

GET /api/feed/following?cursor=&limit=50&viewerId=abc
→ { items: PostCardDTO[], nextCursor }

GET /api/feed/news?cursor=&limit=25&viewerId=abc
→ { items: NewsPostDTO[], nextCursor }

POST /api/reactions
Body: { postId, actorId, type: 'like'|'emoji'|'gif'|'comment'|'share'|'bookmark', emoji?, comment_text? }
→ 200 { ok:true, counts:{...} }

POST /api/flags
Body: { postId, actorId, reason: 'spam'|'abuse'|'misinfo'|'other' }
→ 200 { ok:true }

POST /api/bias-votes
Body: { postId, voterId, vote: 'Neutral'|'LeftBias'|'RightBias' }
→ 200 { ok:true, tallies:{N,L,R,T}, bias_index, neutrality_index, monetized }
```

**DTO hints**

```ts
type PostCardDTO = {
  id, author:{id, handle, display_name, is_news_org},
  kind, title, body, url, created_at,
  engagement:{likes,comments,shares,emojis,gifs,bookmarks},
  flags_count, viewer_has_reacted:boolean
}

type NewsPostDTO = PostCardDTO & {
  bias:{neutral:N, left:L, right:R, total:T, bias_index, neutrality_index, confident:boolean, monetized:boolean}
}
```

## Tests (add unit + integration)

* Ranking math:

  * All: newer high-engagement beats older equal-engagement; heavy flags reduce rank.
  * News: same engagement → higher neutrality\_index ranks higher.
* Bias meter:

  * With votes N=60, L=20, R=20 → neutrality\_index = 0.6 \* (1 - 0) = 0.6 → monetized.
  * With N=10, L=80, R=10 (T=100) → bias\_index = (10-80)/100 = -0.70 → de-monetized.
  * With T=5 → Unrated, no monetization.
* Following:

  * Exact chronological order, stable pagination, no re-ordering.
* Security:

  * Unique constraint on `BiasVote(postId, voterId)`.
  * Rate-limit `POST /api/flags` and `POST /api/bias-votes`.
