Build a module called “Issue Survey” that captures where users stand on major U.S. policy issues in a neutral, structured way.

Goal:
Collect user positions across 30 diverse issues with minimal bias, using consistent, objective phrasing and scalable scoring.

Core Design:
- Route: /issues
- Use ACP Liquid Glass UI
- Present 30 issues, grouped by category
- Each issue = 1 statement + 1 response scale
- Users can skip questions
- Show progress bar
- Save responses for logged-in users

Response Scale (IMPORTANT):
Use a 5-point structured scale focused on LEVEL OF GOVERNMENT INVOLVEMENT:

1 = Strongly Decrease Government Role
2 = Slightly Decrease Government Role
3 = Keep About the Same / Unsure
4 = Slightly Increase Government Role
5 = Strongly Increase Government Role

This keeps all questions objective and consistent.

Also include optional toggle per question:
[ ] “This issue is important to me”

Data Model:
Each issue should include:
- id
- category
- title
- prompt
- description (1 sentence context)
- tags

Store answers as:
{
  issueId,
  response (1–5),
  priority (true/false)
}

Results System:

After completion:
- Show a “Top Issues” list based on priority toggles
- Show “Strongest Positions”:
  - Responses of 1 or 5 = strong stance
- Show “Moderate / Mixed Areas”:
  - Responses of 2–4

Optional Future Feature:
- Match users with candidates based on issue alignment
- Compare user results with district averages

UI Flow:
1. Intro screen:
“Where do you stand on the issues?”
Short explanation + Start button

2. Survey screen:
- Card per issue
- Response slider or buttons
- Priority toggle
- Next/Back navigation

3. Results screen:
- Summary cards
- Top 5 issues (priority)
- Most consistent lean (more 1s vs 5s)
- Download/share option

Important:
- Do NOT label answers as left/right
- Keep language neutral and consistent
- Avoid emotionally loaded wording
- Keep all prompts focused on “level of government involvement”
