Core Philosophy:
ACP does not treat the U.S. as a two-party system. All political parties should be represented equally in the UI. Parties are organizations that may endorse candidates, publish platforms, organize members, and influence elections, but party endorsement alone does not imply corruption.

Candidate Relationship:
Candidates can have many party endorsements.
Parties can endorse many candidates.
Endorsements are informational only.

Do NOT use party ratings, party transparency score, or party controversy index in candidate corruption grading.

Candidate profile integration:
On candidate profiles, show:
“Endorsed By”
- Party logo
- Party name
- Endorsement type
- Source link
- Election cycle

Party Profile Page:
Each party profile should include:

1. Header Card
- Logo
- Name
- Acronym
- Short description
- Website
- Founded year
- Status

2. Political Compass Placement
Show party on same Economic/Social compass system used by ACP.
- Economic axis: -10 to +10
- Social axis: -10 to +10
Label this as “Estimated Platform Position.”

3. Platform Positions
Show issue-by-issue stance connected to ACP Issue Survey categories.
Each row:
- Issue name
- Party stance
- Short summary
- Source

4. Endorsements
Show candidates endorsed by this party.
Filters:
- Election year
- Office level
- State
- Endorsement type

5. Cross-Endorsement Network
Show candidates endorsed by multiple parties.
Example:
“This party shares endorsed candidates with: Green Party, Working Families Party, Forward Party.”

6. User Rating Section
Allow logged-in users to rate the party:
Option A:
Slider from -50 to +50
Labels:
-50 = Strongly Oppose / Distrust
0 = Neutral / Unsure
+50 = Strongly Support / Trust

Optional:
Letter grade A/B/C/D/F

Display:
- Average rating
- Rating count
- Distribution chart
- Do NOT call this a corruption grade

7. Transparency Score
Show party-level transparency score separately:
Score 0–100 based on:
- Funding disclosure clarity
- Platform clarity
- Leadership visibility
- Endorsement source quality
- Ballot access clarity

Label:
“Party Transparency Score”

Tooltip:
“This score reflects how easy it is to verify information about this party. It does not affect candidate corruption grades.”

8. Controversy Timeline
Optional but recommended:
List major controversies, internal conflicts, lawsuits, platform reversals, or scandals.
Each item must require a source URL.
Do not display unsourced claims.

Issue Alignment Feature:
If the user completed the Issue Survey, show:
“Your estimated alignment with this party: X%”

Calculate alignment by comparing:
- user issue survey answers
- party_policy_positions

Simple formula:
For each shared issue:
difference = abs(userStance - partyStance)
issueScore = 1 - (difference / maxDifference)
Average all issueScores and convert to percentage.

Do not call this an endorsement.
Label:
“Estimated Issue Alignment”
