TITLE: Build “Citizen Initiative” (Ballot Measure) Feature for ACP

ROLE
You are a full-stack engineer + product designer. Ship a production-ready, auditable “Citizen Initiative” module that supports proposing measures, gathering signatures, validating eligibility by jurisdiction, and tracking qualification status.

CONTEXT
ACP (Anti-Corruption Party) is a neutral civic platform (social + democracy tooling). This feature lets residents draft a measure, run a signature drive, and progress toward ballot qualification or legislative submission (direct/indirect initiative). Jurisdiction rules vary; build a rules engine + data model to handle state/local differences without code changes.

PRIMARY OBJECTIVES
1) Drafting: create, edit, version, and publish initiative text with title, summary (100–200 words), and full text. Track authors, sponsors, and circulating committee.
2) Jurisdiction rules engine: thresholds, deadlines, circulator rules, direct vs. indirect initiative, county/city scope, and validation heuristics must be data-driven (YAML/JSON).
3) Signature collection: secure petition pages gated by residency/eligibility checks; dedupe and validate signatures; show real-time counts vs. threshold.
4) Validation workflow: automated checks + admin review queue + auditable logs; exportable packets for election officials.
5) Public transparency: initiative profile pages, progress meters, timeline, version history, fiscal-note status, pro/con statements, comment moderation.
6) Security & compliance: KYC-light verification options, rate limiting, salted hashing of sensitive identifiers, and clear consent/attestation language.
